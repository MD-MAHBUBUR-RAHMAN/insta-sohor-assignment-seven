Conceptual Question answer

Javascript is a clientside, interpreted, non-blocking, asynchronous, and concurrent programming language. It runs in a browser. Every browser has its' engine, which parser parses Html elements and CSS style sheets into DOM tree and style rules, respectively. As a single-threaded language, the browser checks javascript code line by line in the parsing process and produces a data structure called Abstract Syntex Text (AST). ATS converts data into machine-readable code, and then code runs on the CPU.

If any changes happen in DOM or style rules browser re-renders the existing layout and reprint on the CPU.

In a browser, javascript code is parsed and executed by its' engine. The engine consists of two parts, a memory heap, and a call stack. In memory-heap, all objects allocate an unstructured way. As a single-threaded non-blocking language, code executes in the browser synchronously. Browser process data according to inside the call stack one after another from its memory heap.
Web APIs such as DOM, AJAX (XML HTTP request), timeout do not exist in a browser. As a result, when the browser tries to read the web API's properties with its other properties, it blocks the data processing way until it processes its previous data.
The Event loop appeared in javascript to solve this problem. The event loop helps the browser to work asynchronously. In the call stack, single data can be processed, and the rest data remain in the callback queue. Data send to the call stack from the callback queue through the event loop. After completing each execution context event loop activated, take data from the callback queue and send it to the call stack. This activity continues till all data end up from the callback queue.
All web API data processes happen when other data are completed.

Web store reserves client-side data. It is also familiar as DOM-storage. A large scale of user data can be stored locally without affecting web application performance and data is never shared to the server.

Data in the web store is stored as a name/value pair by converting them to another format. We can retrieve the data when necessary.

There are two types of web stores that exist in the browser They are: 01.) Local store 02.) session store.

Write window.localStorage to indicate local storage and widow.sessionStorage to indicate session storage.

local storage data is to clear explicitly wheres session storage data is to remove automatically

Stored data do not remove from local storage when the browser is closed but session storage data is lost when a particular session is ended.
